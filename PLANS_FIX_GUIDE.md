# Plans Page - Quick Fix Guide

## المشكلة الحالية
الخطط تُحفظ بدون أسعار وبمدة شهر واحد فقط.

## الحل

### 1. تأكد من ملء الأسعار قبل الحفظ
عند إضافة خطة جديدة:
1. أدخل اسم الخطة
2. اختر المدة (6، 12، أو مخصص)
3. **املأ السعر الشهري (Monthly Price)** - هذا مهم جداً!
4. أضف الخصم إذا أردت
5. سيتم حساب السعر الكلي تلقائياً

### 2. التحقق من البيانات
قبل الضغط على Save، تأكد من:
- ✅ اسم الخطة مملوء
- ✅ المدة محددة (ليست 0 أو 1)
- ✅ السعر الشهري أكبر من 0
- ✅ العملة محددة

### 3. مشاهدة Console Logs
افتح Developer Tools (F12) وشاهد:
```
Submitting plan payload: {
  name: "Professional",
  durationMonths: 12,  // يجب أن تكون 12 وليس 1
  prices: [{
    currency: "IQD",
    monthlyPrice: 35000,  // يجب أن يكون أكبر من 0
    periodPrice: 350000,
    yearlyPrice: 350000,
    discount: 16
  }]
}
```

## التحسينات المضافة

### ✅ Filter Tabs
- All Plans: عرض جميع الخطط
- Active: الخطط النشطة فقط  
- Inactive: الخطط غير النشطة

### ✅ Search
- بحث سريع في أسماء الخطط
- يعمل مع الفلترة

### ✅ Better Initialization
- عند فتح modal جديد، يتم تهيئة الأسعار بشكل صحيح
- المدة الافتراضية 12 شهر

## الخطوات التالية

إذا استمرت المشكلة:
1. افتح Console (F12)
2. جرب إضافة خطة
3. انسخ output الـ "Submitting plan payload"
4. أرسله لي للفحص

## ملاحظات مهمة

- السعر الشهري (monthlyPrice) هو الأساس
- periodPrice و yearlyPrice يُحسبان تلقائياً
- إذا كان monthlyPrice = 0، ستُحفظ الخطة بدون سعر
- إذا كانت durationMonths = 1، تحقق من اختيارك في القائمة
